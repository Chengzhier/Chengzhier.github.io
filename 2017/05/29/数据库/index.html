将数据存储到数据库的优点：方便 效率 安全

关系型数据库
建立在关系模型基础上的数据库，借助于集合代数等数学概念和方法来处理数据库中的数据

在关系型数据库中的名词：
数据库：表的集合
数据表：数据的集合
数据行：一行数据就是一个对象
数据列：也称字段，表示对象的属性
关系数据库的缺点：
1、处理数据格式单一：关系
2、高并发读写性能低：
3、可扩展性低：

是对不同与传统的关系型数据库的数据管理系统的统称

优势：
1.低延迟的读写速度

3.非结构化和不可预知的数据：

比较运算

= > < >= <= !=
select from 表名 where 条件；
1、逻辑判断
AND 指定多个条件，所有条件必须满足
select from 表名 where 条件1 and 条件2；
OR 指定多个条件，满足任意一个条件即可
select from 表名 where 条件1 or 条件2；
NOT 不满足指定条件
select from 表名 where 条件；

模糊查询 关键词 like

%：代表任意多个任意字符
select from 表名 where 要查询的区间 like “要查询的值%”；
_ :
代表一个任意字符，如果多个_则表示多个任意字符
select from 表名 where 要查询的区间 like “_要查询的值”；

固定范围查询

范围查询： IN 表示查询的值指定的集合内
select * from 表名 where 要查询的值 IN(取值的范围);

NOT IN 表示查询的字符不再指定的集合内
select * from 表名 where 要查询的值 NOT IN(取值的范围);

区间范围查询：
between and ：查询的数据在指定的范围区间内，满足条件即可
select * from 表名 where 要查询的值 between 条件1 and 条件2；

not between and 查询的数据不在指定的范围区间内，满足条件即可
select * from 表名 where 要查询的值 not between 条件1 and 条件2；

空值查询
is null ：表示查找数据为空的数据
select * from 表名 where 要查询的值 is null

is not null ：表示查找非空的数据

select * from 表名 where 要查询的值 is not null；

聚合函数

count 统计个数
select count(*) as “别名” from 表名 where 条件；

max 统计某个列里的最大值
select max(要统计的列名) as “别名” from 表名 where 条件；

min 统计某个列里面的最小值
select min(要统计的列名) as “别名” from 表名 where 条件；

sum 统计某一列数据之和，as 可以给查出的数据列起别名

SELECT state, count(*), gender FROM t_hero GROUP BY state HAVING gender=”男”;

SELECT state,count(*) FROM t_hero GROUP BY state;

对select group by 分组之后的结果进行限制输出
having的字段必须
SELECT state,count(*) as num FROM t_hero GROUP BY state HAVING num >3;
