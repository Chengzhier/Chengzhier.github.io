django基于MVC结构的MVT

  • MVC是一种软件设计模式是为了将传统的输入处理输出任务运用到图形化用户交互模型中而设计的，解耦，让不同的代码块之间降低耦合，增强代码的可扩展性和可移植性，实现向后兼容

M ：模型 model 代表应用程序数据，用于控制访问和修改数据业务规则，主要封装对数据层的访问，内嵌ORM框架，实现面向对象的编程来操作数据库，不用考虑数据的差异性，简单配置就可以完成数据库切换
V：控制器 view 提供查询模型相关的状态，用于封装结果，内嵌了模板引擎，实现动态展示数据
C：Controller用来接收get或post请求，处理业务逻辑，与Model和View交互返回结果

Django

M：Model 负责数据处理，内嵌了ORM框架
V：接收HttpRequest业务处理，返回HttpResponse
T:Template,负责封装构造要返回的html，内嵌了模板引擎

T 模板

模板致力于表达外观，而不是程序逻辑

模板的设计实现了业务逻辑view与显示内容template的分离，一个视图可以使用人员一个模板，一个模板可以供给多个视图使用

视图调用模板返回给视图渲染
模板包含两部分
1.静态部分 html css js
2.动态部分 模板语言简写DTL

使用模板的两个过程
1.加载：根据给定的路径找到模板文件，编译后放到内存中
2.渲染：使用上下文数据对模板插值并返回生成的字符串

等同于函数render，直接调用模板

模板语言
模板语言有4种类型

1.变量
语法：

当模板引擎遇到点如book.title的解析顺序：
1.book.title book为名字title为值进行解析book[‘title]先 字典
2.先属性后方法：将book当作对象，查找title如果没有在查找方法title()
3.如果是格式为book.0则解析为列表book[0]
如果变量不存在则插入空字符串
在模板中调用方法时不能传递参数
